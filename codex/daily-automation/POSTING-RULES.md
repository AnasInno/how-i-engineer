# Posting Rules

## Voice

Founder-led. Concrete. No guru slop.

Default identity:

> I'm building TeachClaw as the serious product. On the side, I'm shipping one tiny practical AI automation every day.

## X format

Short and direct:

1. problem
2. what I built
3. why it helps
4. proof / repo link placeholder

Avoid:
- "revolutionary"
- "10x"
- fake virality hooks
- pretending the tool is bigger than it is

## LinkedIn format

Slightly more context:

- who this is for
- the annoying manual workflow
- what the automation does
- what the sample run proves
- limitation / next improvement

## Links

Use placeholders until the repo URL is known:

- `{{REPO_URL}}`
- `{{DEMO_URL}}`
- `{{SCREENSHOT_URL}}`

## Approval

Drafting is allowed. Live posting is blocked unless `scripts/check_approval.py` passes for the exact platform action.
