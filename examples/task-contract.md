# Bounded Task Contract

## Outcome

- one required user-impact outcome
- explicit stop condition

## Scope

- allowed files and systems
- forbidden files and systems
- supported lane-controller actions
- external authority, normally none

## Context

- one relevant skill
- current files
- current evidence
- history explicitly excluded unless required

## Proof

- required surface
- deterministic assertion
- judgement still required
- evidence output
- cleanup responsibility

## Return

- result
- evidence
- verifier status
- cleanup state
- residual risk
